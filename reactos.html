What is ReactOS?

For the past 16 years, ReactOS has been a project to create an OS compatible with the Windows NT kernel, DOS, OS/2 and Java subsystems. According to the developing group, ReactOS is 93.8% complete (alpha). The current build is 0.4.12 released on 09/23/2019 and it reports NT 5.2 (Build 3790; Service Pack 2) on FAT-32. ReactOS can currently run older versions of software originally released for Windows without major glitches. If you get programs from Application Manager (repository), do not upgrade them to the latest version as they might break the OS. For example, the latest version of Firefox in this build is 48.0.2 Extended Support Release (08/24/2016) and 52.0.3 ESR (latest version) would not run. I had to reinstall 48.0.2 ESR to what the installer considered an upgrade (although it was a downgrade) the newer version of Firefox.

Being a Windows clone, it uses the same mentality in naming convention and system variables as Windows. This meant that I had to change the environment variables for TEMP and TMP to C:\TEMP (bad habit that I picked up working with Windows 3.1.1). By the way, I do the same to every Windows machine that I have worked with ever since.

Although the OS is clearly not ready for prime time, I have had fun using it even running a game engine. At this point, I do not know what else to do with it so I do not play with it as much.

ReactOS relies on the Wine project for compatibility with the Windows NT kernel. This is similar to what Unix-like operating systems do. At the same time, this might be its weakest point of the OS although it is a quick way to achieve the goal of making a drop-in replacement for Windows.

Although ReactOS uses Wine, it is not considered and/or marketed as another Unix-like OS. This project is, for lack of a better legal expression, a Windows clone without reusing any of Microsoft's proprietary code that could be used in a fairly new hardware. For make sure the latter remains true, internal audits check the code (mostly C++) constantly.

Installing ReactOS:

Like most operating systems nowadays, the installation of ReactOS is handled by a live .ISO that can be burned to a USB drive. The installation is similar to that of Windows including the dialog boxes. I installed ReactOS in a VDI as I no longer have a physical computer lab at home.

If you decide to use a VDI, you might get errors indicating that some drivers are missing. Some of these errors do not tell you what is missing and/or needed merely displaying that it needs a driver for a system device. If you run it as the sole OS in a computer or shared in a separate partition, you might need to manually install AC97 (available in the Application Manager) for audio and drivers your 802.11 adapter.
